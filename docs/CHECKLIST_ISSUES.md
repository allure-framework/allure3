# Чек-лист проблем

## 1. Parent Launch — суммарное количество тестов на главной странице

**Проблема:** У parentLaunch на main page не отображается суммарное количество тестов.

**Симптомы:** В списке launches для "Parent Launch (Demo Run)" показывается `passed: 0, failed: 0, broken: 0, skipped: 0` и общее количество тестов = 0, хотя дочерние launches содержат тесты.

**Статус:** [x] Исправлено (ListLaunches агрегирует статистику из дочерних launches)

---

## 2. Parent Launch — чужие метаданные

**Проблема:** ParentLaunch загрузил к себе на страничку чьи-то другие метаданные.

**Симптомы:** На странице parent launch отображаются метаданные (ANTIVIRUS_MO, BOOT_TIMEOUT, BUILD_AV, CONFIGURATIO и т.д.), которые относятся к дочерним launches, а не к самому parent launch.

**Статус:** [x] Исправлено (при выборе "All" используются переменные parent launch; убрана авто-подстановка первого child)

---

## 3. Environment — человекочитаемое название вместо ID

**Проблема:** Хочу чтобы у environment отображалось человекочитаемое название (name), а не id.

**Симптомы:** В списке environments показывается UUID (например `4fcf23a6-1511-4cff-b731-aa44e48525ad`), тогда как человекочитаемое имя (`Almalinux8_plesk_selinux_stable_3`, `Ubuntu24_Plesk_500_users_stable_1`) отображается только как вторичная информация ниже.

**Статус:** [x] Исправлено (API возвращает {id, name}; UI показывает name)

---

## 4. Tags — сортировка не работает

**Проблема:** Вижу панель tags, но сортировка по тегам не работает.

**Симптомы:** Панель Tags отображается, можно выбрать тег (например `antivirus`), но фильтрация/сортировка по выбранному тегу не применяется к списку тестов.

**Статус:** [x] Исправлено (теги добавлены в листья дерева; buildFilterPredicate фильтрует по tags)
